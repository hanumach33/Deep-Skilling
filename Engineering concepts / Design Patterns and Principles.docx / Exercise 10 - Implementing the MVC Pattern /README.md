# Exercise 10 - Implementing the MVC Pattern

## Objective

Learn how to implement the Model-View-Controller (MVC) Design Pattern to separate data, presentation, and business logic.

## Scenario

A student management application needs to manage student records while maintaining a clear separation between the model, view, and controller.

## Topics Covered

- MVC Pattern
- Model
- View
- Controller
- Separation of Concerns

## Tasks

- Create the Student model.
- Create the StudentView class.
- Create the StudentController class.
- Update student information through the controller.
- Display student details using the view.
