# Exercise 1 - Implementing the Singleton Pattern

## Objective

Learn how to implement the Singleton Design Pattern to ensure that only one instance of a class exists throughout the application lifecycle.

## Scenario

A logging utility is required for an application. To maintain consistent logging, only one instance of the Logger class should exist.

## Topics Covered

- Singleton Design Pattern
- Private Constructor
- Static Instance
- Static Method
- Object Creation

## Tasks

- Create a Logger class.
- Make the constructor private.
- Create a private static instance.
- Provide a public static getInstance() method.
- Verify that only one object is created.
