# Exercise 5 - Implementing the Decorator Pattern

## Objective

Learn how to implement the Decorator Design Pattern to add notification channels dynamically.

## Scenario

A notification system should support multiple notification channels such as Email, SMS, and Slack. The Decorator Pattern allows adding new notification methods without modifying existing classes.

## Topics Covered

- Decorator Pattern
- Component Interface
- Concrete Component
- Abstract Decorator
- Dynamic Behavior

## Tasks

- Create the Notifier interface.
- Create EmailNotifier.
- Create NotifierDecorator.
- Create SMSNotifierDecorator.
- Create SlackNotifierDecorator.
- Test sending notifications using decorators.
