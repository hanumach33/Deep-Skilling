# Exercise 4 - Implementing the Adapter Pattern

## Objective

Learn how to implement the Adapter Design Pattern to integrate multiple payment gateways with different interfaces.

## Scenario

A payment processing system needs to support multiple third-party payment gateways. Since each gateway exposes different methods, the Adapter Pattern is used to provide a common interface.

## Topics Covered

- Adapter Pattern
- Target Interface
- Adapter Class
- Object Composition
- Polymorphism

## Tasks

- Create the PaymentProcessor interface.
- Create third-party payment gateway classes.
- Create adapter classes.
- Process payments using the adapters.
- Verify the implementation.
