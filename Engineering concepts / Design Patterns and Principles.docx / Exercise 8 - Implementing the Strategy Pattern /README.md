# Exercise 8 - Implementing the Strategy Pattern

## Objective

Learn how to implement the Strategy Design Pattern to select different payment methods at runtime.

## Scenario

A payment system allows customers to choose different payment methods such as Credit Card or PayPal. The Strategy Pattern enables switching between these payment methods without modifying the existing code.

## Topics Covered

- Strategy Pattern
- Strategy Interface
- Concrete Strategies
- Context Class
- Runtime Polymorphism

## Tasks

- Create the PaymentStrategy interface.
- Implement CreditCardPayment.
- Implement PayPalPayment.
- Create the PaymentContext class.
- Select and execute different payment strategies.
- Test the implementation.
